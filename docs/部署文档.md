## ⭐ AdoruWorld 部署完全指南

### 📋 先搞清楚你的项目架构

```
用户浏览器
     ↓
[ Nginx（反向代理）]  ← 推荐，生产环境
  ├── 静态文件（前端构建产物）
  └── /api/* → FastAPI（端口 8000）
                  ↓
          ┌─ adoruworld.db（SQLite）
          └─ uploads/（媒体文件）
```

**当前开发环境的关键配置：**

| 配置项                                         | 当前值                                     |      部署时是否要改       |
| ---------------------------------------------- | ------------------------------------------ | :-----------------------: |
| `client/src/config/index.ts` 的 `BASE_API_URL` | `http://localhost:8000/api/v1`             |          ⚠️ 要改           |
| `server/app/main.py` 的 CORS `allow_origins`   | `["*"]` (全部开放)                         |          ⚠️ 要改           |
| `server/app/config.py` 的 `DATABASE_URL`       | `sqlite+aiosqlite:///...` (环境变量可覆盖) |   ✅ 不用改（环境变量）    |
| `client/vite.config.ts` 的 proxy               | 仅开发环境                                 |  ✅ 不用改（构建后无效）   |
| `server/run.py` 的 uvicorn 启动                | `0.0.0.0:8000`                             | ✅ 部署用 systemd 独立管理 |

### 🏗️ 部署流程

#### 第一步：在服务器上安装环境

```bash
# 1. 安装 conda（如果服务器没有的话）
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh

# 2. 创建环境并安装依赖
conda create -n AdoruWorldServer python=3.11
conda activate AdoruWorldServer
pip install fastapi uvicorn[standard] sqlalchemy aiosqlite pydantic \
            python-multipart pillow mutagen
```

#### 第二步：把代码弄到服务器上

**方式 A — Git 部署（推荐）：**
```bash
# 在服务器上
git clone <你的仓库地址> /var/www/AdoruWorld
cd /var/www/AdoruWorld
```

**方式 B — 用你的导出脚本迁移：**
```bash
# 在本机导出
conda activate AdoruWorldServer
python server/export_all.py -o adoru-backup.zip

# 把 ZIP scp 到服务器
scp adoru-backup.zip user@server:/var/www/AdoruWorld/

# 在服务器上导入
conda activate AdoruWorldServer
cd /var/www/AdoruWorld
python server/import_all.py adoru-backup.zip --clear
```

#### 第三步：构建前端

```bash
cd /var/www/AdoruWorld/client
npm install
npm run build
# → 构建产物在 client/dist/
```

#### 第四步：配置 Nginx 反向代理

```nginx
# /etc/nginx/sites-available/adoruworld
server {
    listen 80;
    server_name your-domain.com;  # ← 改成你的域名

    client_max_body_size 50M;  # 上传文件大小限制

    # 前端静态文件
    root /var/www/AdoruWorld/client/dist;
    index index.html;

    # SPA 路由：所有非文件请求都返回 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 上传文件
    location /uploads/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }
}
```

然后启用：
```bash
sudo ln -s /etc/nginx/sites-available/adoruworld /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

> **如果暂时没有域名**，直接用 IP 也可以，把 `server_name` 改成 `_` 就行。

#### 第五步：配置 systemd 服务让后端自动启动

```ini
# /etc/systemd/system/adoruworld.service
[Unit]
Description=AdoruWorld FastAPI Backend
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/AdoruWorld/server
ExecStart=/home/your-user/miniconda3/envs/AdoruWorldServer/bin/uvicorn app.main:app \
          --host 127.0.0.1 --port 8000 --workers 2
Restart=always
RestartSec=5

# 环境变量（如果用 PostgreSQL 可以在这里设置）
Environment=DATABASE_URL=sqlite+aiosqlite:///var/www/AdoruWorld/server/adoruworld.db
Environment=SITE_START_DATE=2025-01-01

[Install]
WantedBy=multi-user.target
```

> ⚠️ 记得把 `your-user` 替换成你服务器上的用户名！

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl enable adoruworld
sudo systemctl start adoruworld
# 查看状态
sudo systemctl status adoruworld
```

#### 第六步：加上 HTTPS（如果是正式域名）

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```
